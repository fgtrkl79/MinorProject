# MinorProject

Приложение «Цифровой двойник электрической подстанции» — симулятор работы двух трансформаторов с автоматической балансировкой нагрузки, визуализацией данных и управлением аварийными ситуациями.

## Технологии

- .NET 10.0
- Avalonia 11.3.11 (кроссплатформенный UI-фреймворк)
- CommunityToolkit.Mvvm 8.2.1 (MVVM архитектура)
- LiveChartsCore 2.0.0 (графики в реальном времени)
- SkiaSharp (рендеринг графиков)

## Возможности

### Эмуляция физического объекта

Каждый трансформатор симулируется с физически обоснованной моделью:

- **Мощность** — задаётся оператором или балансировщиком, с добавлением шума
- **Температура масла** — рассчитывается на основе мощности, с учётом нагрева и эффекта охлаждения (насосы, вентиляторы), плавное приближение к целевой температуре
- **Напряжение** — случайное блуждание в диапазоне 210–240 В
- **Давление масла** — нормальное значение + шум, обнуляется при протечке

### Автоматическая балансировка нагрузки

Единый слайдер «Общая потребляемая мощность» на главной странице:

- **Три стратегии распределения:**
  - *Т1 → Т2* — Т1 заполняется первым до лимита, остаток на Т2
  - *Т2 → Т1* — Т2 заполняется первым, остаток на Т1
  - *Пропорционально* — делит пропорционально заданным лимитам
- **Порог подключения Т2** — ниже порога Т1 работает один
- **Лимиты балансировщика** — ограничение мощности каждого трансформатора независимо от физического максимума
- **Автоматический failover** — при аварии одного трансформатора вся нагрузка переходит на второй (с предупреждением)

### Ручной режим

На вкладках Т1/Т2 можно включить «Ручной режим» — тогда балансировщик не управляет этим трансформатором, а оператор задаёт мощность слайдером. Остаток мощности распределяется автоматически на второй трансформатор.

### Аварийные ситуации

- **Короткое замыкание** — мгновенный нагрев +40°C, аварийное отключение
- **Протечка масла** — насос работает 10 секунд, затем отключается, температура растёт быстрее
- **Отказ вентиляторов** — вентилятор отключается, охлаждение ухудшается
- **Превышение порогов** — автоматическое отключение при превышении лимита мощности или температуры
- **Починка** — сброс всех аварий, восстановление параметров

### Система охлаждения

Автоматическое управление по температуре:
- **Насос** включается при ≥80°C, выключается при ≤70°C
- **Вентилятор** включается при ≥90°C, выключается при ≤80°C
- Эффект охлаждения учитывается при расчёте целевой температуры

### Отображение данных

- **Графики в реальном времени** — мощность и температура масла (LiveCharts)
- **Таблица показаний** — напряжение, температура, давление, мощность обоих трансформаторов
- **Цветовая индикация** — зелёный (норма), красный (авария), серый (отключён)
- **Предупреждение failover** — при недоступности одного трансформатора
- **Таблица истории** — последние 100 записей с возможностью сортировки
- **Экспорт в CSV** — через системный диалог выбора файла

### Настройки эмуляции

Редактируемые параметры (с валидацией):
- Интервал обновления, максимальная мощность
- Пороги аварийного отключения (мощность, температура)
- Температуры включения/выключения насоса и вентилятора
- Коэффициент нагрева, эффект насоса, эффект вентилятора

Конфигурация сохраняется в `Data/config_t1.json` и `Data/config_t2.json`.

### Смена темы

Переключение светлой/тёмной темы кнопкой в верхней панели. Интерфейс адаптируется через DynamicResource-привязки Avalonia FluentTheme.

## Установка и запуск

1. Установите [.NET 10 SDK](https://dotnet.microsoft.com/download)
2. Откройте терминал в папке проекта
3. Запустите приложение:
   ```bash
   dotnet run
   ```
